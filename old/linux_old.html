Oh, I thought, that probably means you switch off now. No, if you see all the shutdown messages, and it just stops at powerdown, something is wrong: redhat doesn't understand your machine. Is this a problem? Yes. It will corrupt the hard drive occasionaly, and you have to re-install redhat. Zoom ADSL modems

So what next? I go out and buy and ethernet modem, a zoom ADSL X3, and an ethernet card to boot, and SuSE picks it up no problem. So, I am back in business, or so I thought. I wanted to run an Apache web server, so I have to tell the modem to allow me to do this. After many visits to Linux Questions , I still get no joy . Then, I find a help number for the Zoom ADSL on the internet, and after many phone calls, I find someone who has done it on redhat.

The problem was that the manual (all examples in windows XP) is not only badly written, littered with errors, and its instructions for SuSE were out of date, but it's completely wrong! If you want to do port forwarding, here is what you do.

  1. At the command line do /sbin/ifconfig/eth0
  2. Note the inet:addr
  3. Bring up your browser, type in http://10.0.0.2 in the address bar
  4. Username and password are as in the manual
  5. Go to Advanced Setup
  6. Go to Virtual Server
  7. Enter the inet addr number (10.0.0.something) under Host IP address
  8. For http services type in 80 for public and private address
  9. Flash and reboot the modem after save settings
  10. For SuSE, go to YaST-> Security and Users-> Firewall->Allow http access

The manufacturer states that the "something" has to be outside a particular range, and can somehow be any number up to 255, whereas the inet addr:10.0.0.something is inside the range that it says it must not be in. It not only must be a number that is inside the range, but it must be the number given by ifconfig eth0. Also, for DSL connections, ignore all the linux instructions. These instructions might be needed for static IP's, but I think more recent distros will configure it for you anyhow, the SuSE instructions tell you that you must edit a file which no longer exists after SuSE 8.0.

Scanners

Any trouble with the scanners, go to Mier-Geinitz and click on SANE. Read the instructions carefuly, especially the bit on editing the /etc/sane.d/whatever file. For me, whatever is gt68. In my own case the steps are, install the scanner from YaST, which tells you that a firmware upload is needed. Then I download sbfw.usb, and put it in /usr/share/sane/gt68xx. Finally I edit /etc/sane.d/gt68xx.conf and comment out the appropriate override and add a line that says firmware="/usr/share/sane/gt68xx/sbfw.usb".

NVIDIA 3D acceleration

Now this is a pain, the NVIDIA driver is a dummy, with no 3D acceleration. What is more its not that trivial for a newbie to install it. First of all, you have to be in runlevel 3 to install it. Go to /etc/inittab and change the 5 in id:5:initdefault to 3. This means that X windows won't start up on bootup. Get used to the command line, and get the basics of command line editing, I use vi. If you want to start windows after rebooting, you'll have to do "startx" at the command prompt. When you logout, you get back to the command line. To switch the machine off, you type "/sbin/powerd" at the command line (which only works as root). Go to the NVIDIA website, and download, for me at least, NVIDIA-Linux-x86-1.0-5336-pkg1.run. This is a shell script.

Next, you must install the kernel source, which also installs gcc and make. Do an online kernel update too. Now, bring up the command line. If your in the bash shell, change to the c shell (just type csh). You must change to root now to proceed. Type "alias uname='uname -r'", then, "sh NVIDIA-Linux-x86-1.0-5226-pkg1.run" and agree to everything, and finally, type "sax -m 0=nvidia". Note that the NVIDIA-Linux script only runs if windows has not been started, but you have to run sax2 in windows, or it will fail. I got a warning about a conflict with rivafb from the NVIDIA-Linux shell, but everything worked just fine anyway.
